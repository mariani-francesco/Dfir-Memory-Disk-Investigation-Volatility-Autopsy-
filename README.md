# DFIR Memory & Disk Investigation (Volatility & Autopsy)

## Overview

This project demonstrates a Digital Forensics and Incident Response (DFIR) investigation workflow focused on memory and disk forensic analysis.

The lab environment simulates a compromised Windows workstation inside a controlled virtualized environment. The investigation is conducted using industry-standard forensic tools such as Volatility 3 and Autopsy in order to analyze forensic artifacts, identify suspicious activity, reconstruct attacker behavior, and produce a structured technical investigation report.

The project focuses exclusively on the forensic investigation process rather than offensive exploitation techniques.

---

## Objectives

- Perform forensic evidence acquisition
- Conduct memory forensic analysis using Volatility 3
- Analyze disk artifacts using Autopsy
- Investigate suspicious processes and persistence mechanisms
- Reconstruct attacker activity through timeline analysis
- Produce a professional DFIR technical report

---

## Lab Architecture

The environment consists of:

- Victim Machine: Windows 10
- Analyst Machine: Ubuntu / Kali Linux
- VM Platform: VMware Workstation

The victim system is intentionally configured to contain suspicious artifacts and anomalous activity for forensic investigation purposes.
